# Безопасность

Semantic Watch работает только с публичными текстовыми веб-страницами без авторизации.

## Контроль доступа

- доступ разрешается через `TELEGRAM_OWNER_IDS` или ключ активации;
- после активации Telegram ID сохраняется локально;
- ключ сравнивается через constant-time comparison для значений одинаковой длины;
- проверка доступа выполняется до загрузки URL, чтения страниц и LLM-вызовов;
- количество отслеживаемых страниц ограничено для каждого пользователя.

`ACCESS_KEY` является общим ключом активации, а не одноразовым кодом. После выдачи доступа его следует удалить или заменить.

## Защита загрузчика страниц

`SafePageFetcher`:

- разрешает только HTTP/HTTPS;
- запрещает URL со встроенными credentials;
- блокирует localhost, private, link-local и другие non-routable адреса;
- повторно проверяет адрес после каждого redirect;
- фиксирует проверенный DNS-результат для HTTP-соединения;
- ограничивает timeout, размер ответа и число redirect;
- принимает только HTML/XHTML/plain text;
- не отправляет cookies, пользовательские headers и данные авторизации.

В `DEMO_MODE` разрешён только точный `DEMO_URL`. Остальные локальные и приватные адреса остаются запрещены.

## Безопасность AI-пайплайна

- LLM не получает tools и не может загружать страницы, менять данные или отправлять команды;
- AI вызывается только при формировании правила и при обновлении hash страницы;
- правило показывается пользователю и сохраняется только после подтверждения;
- размер текста страницы и diff ограничен;
- число LLM-вызовов ограничено суточным лимитом;
- JSON-ответ модели валидируется через Zod;
- совпадение принимается только при `conditionMatched=true` и достаточной уверенности;
- каждая цитата `evidence` должна буквально присутствовать в текущем тексте страницы;
- при ошибке API, невалидном JSON или выдуманном evidence новый snapshot не сохраняется;
- fingerprint предотвращает повторную отправку одного результата.

Текст страницы может содержать prompt injection. Риск снижен структурированным ответом, отсутствием tools и детерминированной проверкой evidence, но ошибочная классификация полностью не исключается.

## Scheduler и доставка уведомлений

- одну страницу нельзя одновременно проверять через scheduler и вручную;
- приостановленные страницы не участвуют в фоновых проверках;
- число проверок за один scheduler tick ограничено;
- после ошибки следующая попытка откладывается;
- совпадение сохраняется как pending notification до успешной отправки в Telegram;
- при ошибке Telegram уведомление не помечается доставленным;
- число автоматических попыток доставки ограничено;
- пока уведомление ожидает доставки, новое состояние страницы автоматически не проверяется;
- удаление страницы удаляет ожидающее уведомление.

## Секреты и данные

- Telegram token, DeepSeek key и ключ активации загружаются из environment variables;
- `.env`, `data/*.json`, `dist/` и `node_modules/` не коммитятся;
- JSON-хранилище записывается через временный файл и atomic rename;
- локальное хранилище не шифруется, поэтому доступ к каталогу приложения должен быть ограничен средствами ОС.

## Ограничения MVP

Не поддерживаются:

- страницы с авторизацией;
- cookies и пользовательские headers;
- browser automation;
- JavaScript-rendered страницы;
- PDF, изображения и визуальное сравнение;
- доступ к private networks;
- автоматические действия на отслеживаемых сайтах;
- rate limiting попыток ввода ключа активации;
- шифрование локального JSON-хранилища.

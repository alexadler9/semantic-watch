# Semantic Watch

Telegram-сервис, который отслеживает нужную информацию на публичных веб-страницах.

Пользователь добавляет страницу и описывает, что должно на ней появиться или произойти. AI преобразует описание в понятное правило, пользователь подтверждает его, после чего сервис периодически проверяет страницу и сообщает только о подходящем результате.

## Как работает

```text
пользователь добавляет URL
→ описывает, что нужно отслеживать
→ AI при необходимости задаёт уточняющий вопрос
→ AI формирует структурированное правило и текущее смысловое состояние
→ пользователь подтверждает правило
→ scheduler проверяет страницу
→ hash не изменился: AI не вызывается
→ текст обновился: diff + прошлое смысловое состояние + AI-анализ
→ проверка confidence и evidence
→ Telegram-уведомление
→ пользователь подтверждает результат или уточняет правило
```

AI:

- интерпретирует пользовательское описание;
- задаёт уточняющий вопрос, если задача допускает несколько трактовок;
- формирует правило и кратко описывает текущее состояние страницы;
- определяет, появилась ли нужная информация;
- предлагает причины, почему найденный результат мог не подойти;
- уточняет правило по обратной связи пользователя.

Обычный код отвечает за доступ, безопасную загрузку страниц, расписание, лимиты, хранение, валидацию AI-ответа и доставку уведомлений.

## Стек

- Node.js 20+
- TypeScript
- grammY
- Cheerio
- DeepSeek API
- JSON storage с атомарной записью
- Telegram Bot API через long polling

## Конфигурация

Создайте `.env` на основе `.env.example`:

```env
TELEGRAM_BOT_TOKEN=...
TELEGRAM_OWNER_IDS=123456789
ACCESS_KEY=

DEEPSEEK_API_KEY=...
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash

SCHEDULER_ENABLED=true
SCHEDULER_TICK_SECONDS=60
DEFAULT_CHECK_INTERVAL_MINUTES=30
MAX_LLM_CALLS_PER_DAY=50
```

Доступ разрешается владельцам из `TELEGRAM_OWNER_IDS`. Альтернативно можно задать общий `ACCESS_KEY` и один раз выполнить:

```text
/activate <ключ>
```

После активации Telegram ID сохраняется в `data/store.json`.

## Локальный запуск

```powershell
pnpm install
pnpm run typecheck
pnpm start
```

Демонстрационная страница запускается в отдельном терминале:

```powershell
pnpm run demo:closed
pnpm run demo
```

Адрес:

```text
http://127.0.0.1:3001/
```

Доступные состояния:

```powershell
pnpm run demo:closed
pnpm run demo:speakers
pnpm run demo:partners
pnpm run demo:waitlist
pnpm run demo:open
```

- `closed` — регистрация закрыта;
- `speakers` — обновился состав спикеров;
- `partners` — регистрация доступна только партнёрам и приглашённым;
- `waitlist` — открыт только лист ожидания;
- `open` — регистрация доступна всем участникам.

## Фоновые проверки

Scheduler:

- выбирает активные страницы, у которых наступило время проверки;
- не запускает ручную и фоновую проверку одной страницы одновременно;
- повторяет временно неудачные проверки позже;
- хранит найденный результат до успешной доставки в Telegram;
- не проверяет следующее состояние страницы, пока уведомление ожидает доставки;
- не проверяет приостановленные страницы.

## Данные и безопасность

Не коммитятся:

- `.env`;
- `data/*.json`;
- `dist/`;
- `node_modules/`.

AI вызывается при настройке и уточнении правила, формировании вариантов обратной связи и при реальном обновлении текста страницы. Ответы модели валидируются схемами, а найденный результат дополнительно проверяется порогом уверенности и точными цитатами из текущего snapshot.

Подробнее: [SECURITY.md](SECURITY.md).

## Ограничения MVP

Не поддерживаются страницы с авторизацией, cookies, JavaScript-rendering, PDF, изображения и автоматические действия на отслеживаемых сайтах.

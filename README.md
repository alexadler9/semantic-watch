# Semantic Watch

Telegram-сервис для мониторинга публичных веб-страниц.

Пользователь задаёт URL и описывает, на какие изменения нужно обратить внимание. Сервис сохраняет исходное состояние страницы, периодически проверяет её и отправляет уведомление только тогда, когда AI подтверждает нужное изменение.

## Как работает

```text
Telegram-запрос
→ безопасная загрузка страницы
→ snapshot и SHA-256 hash
→ фоновая проверка по расписанию
→ hash не изменился: AI не вызывается
→ текст изменился: diff + AI-анализ
→ проверка confidence и evidence
→ Telegram-уведомление
```

AI:

- преобразует пользовательское описание в правило наблюдения;
- оценивает изменение страницы;
- формирует краткое объяснение.

Обычный код отвечает за доступ, загрузку страниц, расписание, лимиты, хранение, проверку ответа модели и отправку уведомлений.

## Конфигурация

Создайте `.env` на основе `.env.example`:

```env
TELEGRAM_BOT_TOKEN=...
TELEGRAM_OWNER_IDS=123456789
ACCESS_KEY=

DEEPSEEK_API_KEY=...
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash

SCHEDULER_ENABLED=true
SCHEDULER_TICK_SECONDS=60
DEFAULT_CHECK_INTERVAL_MINUTES=30
MAX_LLM_CALLS_PER_DAY=50
```

Доступ разрешается владельцам из `TELEGRAM_OWNER_IDS`. Альтернативно можно задать общий `ACCESS_KEY` и один раз выполнить:

```text
/activate <ключ>
```

После активации Telegram ID сохраняется в `data/store.json`. Если новые активации больше не нужны, удалите `ACCESS_KEY` и перезапустите сервис.

## Локальный запуск

```powershell
pnpm install
pnpm run typecheck
pnpm start
```

Демонстрационная страница запускается в отдельном терминале:

```powershell
pnpm run demo:closed
pnpm run demo
```

Адрес:

```text
http://127.0.0.1:3001/
```

Состояния страницы:

```powershell
pnpm run demo:closed
pnpm run demo:speakers
pnpm run demo:open
```

## Команды бота

```text
/start
/activate <ключ>
/watch
/list
/check [ID]
/stop <ID>
/cancel
```

`/check` запускает внеплановую проверку. Без этой команды активные задачи проверяются автоматически.

## Фоновые проверки

Scheduler запускается внутри процесса приложения:

- выбирает задачи, у которых наступило время проверки;
- не запускает одну проверку одновременно из scheduler и `/check`;
- повторяет неудачные проверки позже;
- хранит уведомление до успешной доставки в Telegram;
- не проверяет следующее состояние страницы, пока важное уведомление ожидает доставки;
- ограничивает число проверок за один проход.

Параметры находятся в `.env.example`.

## Данные и безопасность

Не коммитятся:

- `.env`;
- `data/*.json`;
- `dist/`;
- `node_modules/`.

AI вызывается только при создании задачи и при реальном изменении текста страницы. Ответ модели валидируется схемой, порогом уверенности и наличием точных цитат в текущем snapshot.

Подробнее: [SECURITY.md](SECURITY.md).

## Ограничения MVP

Не поддерживаются страницы с авторизацией, cookies, JavaScript-rendering, PDF, изображения и действия на отслеживаемых сайтах.
